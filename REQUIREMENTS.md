# 自作IDE 要件定義

## 背景

現状の開発環境（cmux + Ghostty + yazi + git-watch + Claude Code）から、自作のオリジナル開発環境に移行したい。Mac でだけ使えればいい。

**自分専用ツール**として作る。配布・署名・自動更新・他ユーザー対応は考慮しない（ローカルビルドのみで運用）。

### 想定スタック（PoC 通過前提）
- **Swift + libghostty + SwiftUI**
- ターミナル: **libghostty** を組み込んで Ghostty 設定（`~/.config/ghostty/config`）を継承
- ファイル系UI（ツリー・検索・プレビュー）: SwiftUI
- シンタックスハイライト: NSTextView + tree-sitter（または Highlightr）
- 参考実装: [cmux](https://github.com/manaflow-ai/cmux) のソース

PoC で libghostty 統合が詰まった場合は **Tauri + xterm.js + CodeMirror 6 + portable-pty** にフォールバック。

参考: [Cursorからcmux/Claude Codeへの移行構成](https://zenn.dev/d0ne1s/articles/7adbd3a3d54b1d)

## 1. 全体レイアウト

```
┌──────────┬─────────────────────┬─────────────────┐
│          │ [←][→][🌲ツリー]   │  上小ターミナル │
│ プロジェ │ ─────────────────── │ （タブ複数）    │
│ クト一覧 │ ファイルツリー      │                 │
│ サイド   │   ⇕（クリック切替） ├─────────────────┤
│ バー     │ ファイルプレビュー  │  下大ターミナル │
│          │                     │ （タブ複数）    │
└──────────┴─────────────────────┴─────────────────┘
```

- **左サイドバー**: プロジェクト一覧
- **中央ペイン**: ファイルツリー ⇔ ファイルプレビューがクリックで切り替わる。**切替時にサイドバーとターミナルは影響を受けない**（中央ペインだけ差し替わる）
  - 上部にツールバー: 戻る/進むボタン、ツリーに戻るボタン
- **右ペイン**: 上小ターミナル + 下大ターミナル、各々が独立したタブ群を持つ

### ペイン比率
- ドラッグでリサイズ可
- 比率の保存はしない（外部ディスプレイと MacBook で都度調整する想定）

### 廃止する要素
- **git-watch ペイン**（最近使ってないため）
- **yazi**（自作のVSCode風ファイルツリーで代替）

## 2. プロジェクト管理モデル

### 想定スケール
- **ピン留めプロジェクト（永続）**: 10〜15個程度
- **同時に開く（メモリ上にロード）**: 3〜5個程度
- 想定環境: 32GB RAM

### サイドバーの構造（cmux 風）

```
┌─────────────────┐
│ ＋ 新規追加     │
├─────────────────┤
│ 📌 project-A    │  ← ピン留め（手動並び替え）
│ 📌 project-B    │
│ 📌 project-C    │
├─────────────────┤
│    tmp-D        │  ← 一時プロジェクト（MRU順）
│    tmp-E        │
└─────────────────┘
```

### プロジェクトを追加する
- サイドバー上部の **「＋」ボタン → フォルダ選択ダイアログ** で追加
- 追加直後は**一時プロジェクト**としてサイドバー下部に表示される
- ユーザーが**ピン留め**することで永続化される

### ピン留め
- ピン留めしたプロジェクトのみアプリ再起動後も**サイドバーに残る**
- ピン留めは手動並び替え可能（ドラッグ）
- 一時プロジェクトはピン留め群の下に **MRU 順** で並ぶ

### 一時プロジェクト
- アプリ終了で**消える**
- 明示的に「閉じる」操作もあり（右クリックメニュー等）

### 「開く」操作（=メモリにロード）
- サイドバーからプロジェクトを選択すると開く
- 初回オープン時にファイルツリーをロード、ターミナルセッションを起動
- 2回目以降の切替は表示の切り替えだけ（インスタント）
- ターミナルセッションは生きっぱなし（claude code を回しっぱなしにしておける）

### アプリ再起動時
- ピン留めプロジェクトはサイドバーに復元される（一時プロジェクトは消える）
- 各プロジェクトのUI状態（開いてたファイル、ターミナルセッション、ファイルツリーの開閉状態）はリセット
- 再度サイドバーから選んだタイミングでロードされる

### 永続化スキーマ
ピン留めプロジェクト等の状態を永続化する先:

- 保存先: **`~/Library/Application Support/<app名>/projects.json`** （macOS 標準）
- 形式: JSON、スキーマバージョン番号付き
- 書き込み: アトミック書き込み（一時ファイル → rename）
- 破損時: 直近のバックアップから復旧（数世代保持）
- 設定リセット手段: メニューから「設定をリセット」or 該当ファイル削除で初期化

### プロジェクトパスが消えたときの挙動
ピン留め済みプロジェクトのパスが消えた・権限ない・外部ディスク未接続の場合：

- サイドバーで **「missing」状態を表示**（薄字＋ ⚠ アイコン等）
- クリックしても**開けない**（エラートースト等で通知）
- 一時的なマウント解除でもピン留めは**消さない**（再接続すれば復活）
- 右クリックメニューに「再選択（パスを変更）」「ピン解除して削除」を用意

### 初回起動 UX
ピン留めプロジェクトがゼロのときの状態：

- サイドバーは**空状態**で、上部の「+」ボタンを**目立たせる**
- 中央ペインに「フォルダを追加して始めよう」程度の**案内テキスト**を表示
- 一度プロジェクトを追加したら案内は消える

### メモリ管理（将来検討）
- MVP では手動アンロード手段は持たない
- 32GB RAM 環境で同時5プロジェクト程度なら 2〜3GB 程度のメモリ消費で問題なし
- もし運用上重くなることがあれば、サイドバー右クリックで「アンロード」（メモリから降ろす、ターミナルプロセスも kill）を追加する余地あり

## 3. プロジェクト切替（Ctrl+M）

[tab-trace](https://github.com/d0ne1s/tab-trace) と同じ MRU（最近使った順）の挙動。

### 操作
- **Ctrl+M**: オーバーレイ起動 + 直前プロジェクトにカーソル
- **Ctrl 押しっぱなしで M 連打**: MRU リストをサイクル（直近5件程度）
- **Ctrl 離す**: 確定
- **Esc**: キャンセル

### 仕様
- IDE ウィンドウ内でのみ捕捉（OS全体ではない）
- ターミナル含む全エリアで動作
- ターミナルにフォーカスがあっても Ctrl+M は IDE が最優先で捕捉
- **TUI アプリ（vim, fzf, less, claude code 等）の中でも例外なく IDE が捕捉**
- **逃がし手段は持たない**（自分専用ツールなので意図的な制約として固定）
- **Enter は Return キーで打つ**運用（Ctrl+M を Enter として使わない）
- MRU スタックに入れるのは **プロジェクトのみ**（ファイルやタブは含めない）
- ピン留め・一時プロジェクトは**区別せず**、開いてるプロジェクトはすべて MRU の対象

### MRU 更新のタイミング
- **表示確定時に更新**（Ctrl 離して確定した瞬間）
- **Esc キャンセル時は不変**（元のスタックのまま）
- **同一プロジェクトの再選択は no-op**（順序変えない）
- サイドバークリックで開いた場合も同様（実際にプロジェクトが切り替わったタイミングで更新）

## 4. ターミナル

### 構成
- 上小ターミナル + 下大ターミナル
- 上ペイン・下ペインそれぞれが**独立したタブ群**を持つ
- 各ペイン内でタブ複数

### 起動時の挙動
- cwd は**プロジェクトルート固定**
- プロジェクトを最初に開いたとき: **上下とも空シェル1タブ** で起動
  - claude code や codex は手動で起動するスタイル

### タブ機能（最小セット）
- **追加（空シェル）**: Cmd+T または「+」ボタン
- **閉じる**: Cmd+W または「×」ボタン
- **タブジャンプ**: Cmd+1〜9 でフォーカスがあるペインの N 番目のタブに切替
- **タブのタイトル**: cwd（または実行中プロセス）から**自動表示**、手動リネームは持たない
- **並び替え**: 持たない（必要になったら将来追加）

### タブを閉じる時の確認（PTY 動作中）
- foreground プロセスが **shell 以外**（claude / codex / 長時間コマンド等）の場合、Cmd+W や ×ボタンで閉じる際に **確認ダイアログ**を表示
- 「本当に閉じる？プロセス XXX が動作中です」のような注意喚起
- shell プロンプトに戻っている場合は確認なしで即閉じる
- **判定方法（割り切り）**: foreground process group が shell のみなら確認なし。background job（`&` で動かしてるプロセス等）は検知しない

### shell・PTY 異常終了時の挙動
- shell や子プロセスが異常終了した場合、**タブ内に「終了しました（exit code: N）」**を表示
- **再起動ボタン**を表示（クリックで新しい shell プロセスを起動）
- タブは自動では閉じない（ユーザーが exit code やエラーログを見られる）
- ビルドコマンドが失敗した時の出力なども見たまま残せる

### shell の起動方式
- IDE 内 PTY は **`$SHELL -l` で起動**（ログインシェル）
- 自分の環境では zsh 前提（`.zprofile` + `.zshrc` 両方読み込み）
- `$SHELL` が他のシェル（fish/bash 等）の場合もログインシェルとして起動するので、各シェルの起動ファイルが読まれる
- PATH や環境変数が Ghostty 起動時と同じになり、既存の Claude Code カスタムスキル・hooks がそのまま動作する前提を守る

### AI種別バッジ（タブ表示）
タブで動いている AI を視覚的に区別するためのバッジ。
- **Claude が動いてるタブ**: 🅒 アイコン（または専用カラー）
- **Codex が動いてるタブ**: 🅞 アイコン
- **空シェル**: バッジなし
- **検知方法**: PTY の foreground プロセス監視で自動検知。プロセスが終了したらバッジは消える
- AI を起動するときは shell で手動で `claude` / `codex` を打つ（起動ショートカットは MVP では持たない）

### ペイン間のフォーカス移動
- **マウスクリックのみ**（キーバインドは持たない）

### ターミナル出力中のパス/URL検出（クリック可能化）
- **ファイルパス**（例: `path/to/file.ts:42:8`）: クリックで IDE のファイルプレビューに開く（行番号付きならその行にジャンプ想定）
- **URL**: クリックで外部ブラウザで開く
  - **`http` / `https` のみリンク化**（`file://` やカスタムスキームは無視）
- Claude Code の出力との連携を意識した機能

### その他標準動作
- マウス選択でコピー、Cmd+C / Cmd+V
- スクロールバックはデフォルト値（数千行）
- フォント・カラースキームは将来カスタマイズ可

## 5. AIワークフロー（Lv1: 並列ターミナル運用）

### やりたいこと
- 壁打ち → プランファイル作成 → 別AIで評価 → 実装 → 実装結果を別AIで評価
- **Claude Code と Codex CLI を両方使える**ようにする

### 想定ワークフロー
```
[壁打ち] → [プランファイル作成] → [Claude実装] → [Codexレビュー] → [Claude修正] → ...
                                       ↓ 落ちた / 詰まった
                                  [Codexに逃がす]
```

役割分担の例：
- **Claude Code**: 設計・実装・修正
- **Codex**: 設計レビュー・実装レビュー・障害時のフォールバック

### 方針
- 別タブで別AIエージェントを起動するだけ。プランファイル（`docs/plans/xxx.md`）を介して情報を共有
- IDE 側で特別なオーケストレーション機能は持たない（ターミナルから普通に CLI を起動するだけ）
- チャットUI（Codex app 風）は Claude を API 経由にする必要があるため**諦め**。ターミナル経由を採用

### MVP に含める AI 補助機能
- **タブの AI 種別バッジ**: タブに動作中の AI（Claude / Codex）をアイコン表示（詳細はセクション 4 参照）
- **AI 完了のバッジ通知**: BEL 検知でサイドバー＋タブにバッジ表示（既出）

### 将来検討（MVP では実装しない）
- **AI起動ショートカット**: 固定3択（Claude / Codex / 空シェル）を Cmd+Shift+T で起動
- **カスタム起動テンプレート**: プロジェクト別に「上=shell、下=claude+codex」のような構成を保存
- **プロンプトテンプレートのワンタッチ挿入**（設計 / レビュー / 実装などのテンプレを Cmd+Shift+数字 で挿入）
- **プランファイル連携**（`docs/plans/` 認識、パス挿入、チェックボックス進捗可視化）
- **AI ヘルス監視**（Claude/Codex 障害検知）
- **クロスエージェント引き継ぎ補助**（最後の会話を別 AI に貼る支援）
- **BEL 通知の誤検知対策**（AI種別バッジ付きタブのみに絞る、debounce、手動クリア等）

### AI完了のバッジ通知

cmux と同様、AI のアウトプットが終わったタイミングでサイドバーやタブにバッジを表示する。

#### 検知方式（A + B の組合せ）
- **IDE側**: libghostty のターミナル出力ストリームから **BEL (0x07)** を聞き取る
- **ユーザー側**: Claude Code の Stop hook や zshrc に `tput bel` を仕込んで完了時に BEL を発生させる
- IDE 実装はシンプル、振る舞いはユーザーの hook 設定で柔軟に変更可

#### バッジ表示の粒度
- **プロジェクト単位**: サイドバーの該当プロジェクトに点灯
- **タブ単位**: 上下ペインの該当ターミナルタブにも点灯
- **非アクティブ時のみ点灯**: 自分が現在見ているプロジェクト・タブにはバッジは出さない
- **クリア条件**: 該当プロジェクト/タブをアクティブにしたら自動で消える

## 6. ファイル・ディレクトリ操作

### 6.1 Cmd+P クイック検索（ファイル名・パス）

VSCode の Cmd+P 同等の挙動。

#### 検索範囲・アルゴリズム
- スコープ: **アクティブプロジェクト内のみ**
- ファジーマッチ（`uc` で `UserController.ts` がヒット）
- 入力に **スラッシュを含むとパスマッチに自動切替**（`src/utils/foo` でパス検索）
- スコアリング: 直近開いたもの・ファイル名一致を上位に
- 検索対象: **ファイル + ディレクトリ**
  - ディレクトリを選んだ場合はファイルツリーでそこへジャンプ
  - ファイルを選んだ場合は全画面プレビューで開く

#### インデックス・検索ポリシー（ツリーと統一）
- プロジェクト初回オープン時に全ファイル一覧をインデックス
- ファイルシステム watcher で増減を反映
- **隠しファイル**（`.git`, `.env` 等）: **デフォルトで含める**（ツリーと同じ）
- **`.gitignore` 対象**（`node_modules`, `dist` 等）: **デフォルトで除外**（VSCode と同じ）
  - 検索 UI に **「ignored を含む」トグル**を用意し、必要時に切替可能

#### キーバインド
- **Enter**: 開く（ファイル → 全画面プレビュー、ディレクトリ → ツリーへジャンプ）
- **Cmd+C**: 選択中項目のパスをコピー
- **↑↓**: 選択移動
- **Esc**: キャンセル

### 6.2 Cmd+Shift+F 全文検索（grep）

VSCode と同等。**ripgrep** を内部で使用。

- スコープ: アクティブプロジェクト内のみ
- 検索ポリシーは Cmd+P と同じ（隠しファイル含める／`.gitignore` 対象は除外、トグルで切替可）
- 結果一覧から該当行へジャンプ可能
- Cmd+P と同じく Cmd+C でパスコピーできる想定
- **結果量・キャンセル制御**:
  - 結果**上限 1000 件**（超えた分は省略表示、「もっと見る」も持たない）
  - **10 秒タイムアウト**
  - 検索中は **Esc でキャンセル** 可能

### 6.3 ファイル/ディレクトリパスのコピー

以下のいずれの場面でもパスを楽にコピーできるようにする：
- Cmd+P・Cmd+Shift+F の検索結果から（Cmd+C）
- ファイルツリーから（右クリックメニュー or キーバインド、詳細は未確定）

### 6.4 ファイルプレビュー（編集なし / Phase 0 採用）

#### 方針
- **閲覧専用**。IDE 内では編集しない
- 編集したいときは **外部 VSCode を起動**して移行
- Phase 1 以降への拡張は将来の課題

#### 表示モデル
- **シングルファイル表示＋戻る/進む履歴ナビゲーション**（ブラウザ風）
- VSCode 風の複数タブは持たない
- ファイルツリーで別ファイルをクリックすると差し替え、履歴が残る

#### 中央ペインの切替挙動
- ファイルツリーで**ファイルをクリックするとプレビューに切り替わる**（中央ペインのみ。サイドバー・ターミナルは影響を受けない）
- **ツリーに戻る方法**:
  - **Esc キー**
  - 中央ペイン上部のツールバーにある **「ツリーに戻る」ボタン**
- **戻る/進む履歴ナビ**:
  - 中央ペイン上部のツールバーの **← →ボタン**で操作（マウスのみ、キーバインドなし）

#### 各形式の表示
| 形式 | 表示方法 |
|---|---|
| コード（汎用） | **NSTextView + tree-sitter**（または Highlightr）でシンタックスハイライト付き読込専用表示 |
| Markdown | **レンダリング済みプレビューのみ**（grip/mo の代替） |
| 画像（png/jpg/gif/svg/webp） | 標準ビューワー |
| PDF | 標準ビューワー（あるいは外部アプリ起動） |
| その他 | プレーンテキスト or 「外部アプリで開く」誘導 |

#### サイズしきい値
| サイズ | 挙動 |
|---|---|
| 〜5MB | そのままプレビュー |
| 5MB 超 | 「開きますか？」確認ダイアログ |
| 50MB 超 | 外部アプリ起動を誘導（IDE 内では開かない） |

#### 表示中ファイルが外部削除された場合
- 中央ペインに **deleted 状態**を表示（「このファイルは削除されました」等）
- 戻る/進む **履歴は保持**（再作成されたら復活して同じ位置で表示できる）
- Claude Code がファイルを作り直したり書き換えたりするケースを想定

#### バイナリ・エンコーディング不明ファイル
- ファイルの先頭バイトを読んで判定
- **NUL 文字を含む**場合は**バイナリ扱い**（プレビューしない、外部アプリ誘導）
- **UTF-8 デコードに失敗**した場合（壊れたテキスト・UTF-16・Shift_JIS 等）も**外部アプリ誘導**
- 自動でエンコーディング判定はしない（コスト対リターンが見合わない）

#### 外部変更への対応
- **常に自動リロード**（fs watcher が変更を検知したら即座に再読み込み）
- 閲覧専用なので未保存状態が発生せず、競合の懸念なし
- Claude Code がファイルを編集したら IDE 側でもすぐに最新版が見える
- **閲覧位置の維持**:
  - **行番号基準**でスクロール位置を維持（読んでた箇所を見失わない）
  - 元の行が削除された場合は**近傍行**にフォールバック

#### 外部 VSCode 起動
- ショートカット（例: **Cmd+Option+O**）で現在表示中のファイルを VSCode で開く
- ファイルツリーの右クリックメニューにも「VSCode で開く」を用意

## 7. ファイルツリーの仕様

中央ペインに表示する VSCode 風のファイルツリー。yazi の置き換え。

### 基本方針: 閲覧専用
- **CRUD 操作（新規作成・リネーム・削除・複製・D&D 移動）は実装しない**
- ファイル/ディレクトリ操作はすべて**ターミナル経由**（mv, rm, cp, mkdir 等）で行う
- ツリーはファイルシステム watcher で外部変更を自動反映

### ファイル監視の取りこぼし対策
fs watcher は巨大リポジトリや短時間大量変更で取りこぼしが発生しうる。

- **手動更新ボタン**をツリー上部に用意（F5 相当）
- **定期的な差分再スキャン**（60秒程度の間隔）で取りこぼしを検出
- watcher が overflow した場合は次回スキャン時に自動で全更新

### シンボリックリンクの扱い
ツリー・Cmd+P インデックス・grep・watcher で統一：

- **ディレクトリ symlink は辿らない**（循環リスク回避）
- **ファイル symlink は表示する**（読込み時にリンク先を解決）
- **プロジェクトルート外を指す symlink は明示表示**（アイコン or 色変更）

### 表示・並び順
- **フォルダ先・アルファベット昇順固定**（並び替えオプションは持たない）
- ファイルアイコン（拡張子別の標準アイコンセット）

### 隠しファイル・.gitignore の扱い（Cmd+P と統一）
- **隠しファイル（`.git`, `.env` 等）はデフォルト表示**
- **`.gitignore` 対象（`node_modules`, `dist` 等）は薄く表示**
- 完全非表示にするトグルあり
- 検索系（Cmd+P / Cmd+Shift+F）でのポリシーはセクション 6 と同じく統一されている

### git status バッジ
git-watch 廃止の代替として、ファイルツリー上に表示。

- **シンプル版**: ファイル単体に色付け＋文字バッジ（M/A/D/?/!!）
- ディレクトリ伝播表示は **MVP では実装しない**（実装コストとリターン次第で将来追加検討）
- **更新トリガー**:
  - **プロジェクト fs watcher** の変更イベントが**主トリガー**（未追跡ファイルや作業ツリー変更も拾える）
  - `.git/index` watcher は staging / checkout 系の補助トリガー
- メモリ消費は数百ファイル分でも微小
- **頻度・キャンセル制御**:
  - **200ms debounce** で連続更新をまとめる
  - 前回実行中は完了を待ってからスキップ判定（最新の1回だけ実行）
  - **10秒タイムアウト**で大規模 repo でも止まらない

### 補助操作（右クリックメニュー）
- **相対パスをコピー**（プロジェクトルートからの相対パス）
- **ターミナルで開く**（右ペインのターミナルに新タブを作成し、cwd をそのディレクトリに設定）

### 検索
- ツリー内のインクリメンタル検索（Cmd+F）は持たない
- ファイル探索は **Cmd+P** に集約

## 8. 横断的関心事項

### 8.1 外部コマンド起動の安全性
VSCode起動・ターミナルで開く・AI起動など、外部コマンドにパス等を渡す箇所での事故防止：

- **shell 文字列結合は使わず、argv 配列で起動**（`spawn(cmd, [arg1, arg2, ...])` 形式）
- スペース・引用符・改行・制御文字を含むパスでも事故が起きない実装を要件化
- shell 経由が必要な場合のみ shell-quote 等で適切にエスケープ

### 8.2 ログ・診断
PTY・watcher・git status・ripgrep・外部アプリ起動など失敗しうる箇所のログ：

- 保存先: `~/Library/Logs/<app名>/` （macOS 標準）
- ログレベル設定（error / warn / info / debug）
- メニュー or ショートカットから**直近のログを開く**機能
- ログファイルはローテーション（数日分保持、上限サイズ）
- パス等の個人情報を含むため、共有時の注意喚起をUIに含める

### 8.3 エラー表示の場所・粒度
エラー種別ごとに出し分ける：

| 種別 | 表示先 | 例 |
|---|---|---|
| **操作起因の単発エラー** | toast | VSCode 起動失敗、grep 失敗、パスコピー失敗 |
| **継続的な状態異常** | サイドバー or 該当ペイン内に常駐 | watcher 停止、ピン留めパスが missing、PTY異常終了 |
| **詳細情報** | ログファイル | スタックトレース、外部コマンドの stderr |

### 8.4 同期ストレージ（Dropbox 等）配下のプロジェクト
Dropbox 配下のディレクトリをプロジェクトとして扱うことがある。

- 通常のディレクトリと同じく扱う（特別なロジックは持たない）
- 同期遅延・一時的なファイル消失は既存の **missing / deleted / watcher 取りこぼし対策**でカバー
- Dropbox の競合ファイル（`(競合 ...)` 等）はツリーに表示されるが特別扱いしない
- Smart Sync の placeholder ファイルも標準の File Provider API 経由で自動ダウンロードされる前提
